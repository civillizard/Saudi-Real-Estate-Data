# بيانات العقار المفتوحة في المملكة العربية السعودية
## Saudi Real Estate Open Data

[![License: MIT](https://img.shields.io/badge/code-MIT-blue.svg)](LICENSE)

🌐 [English version](README.md)

بيانات مفتوحة موحّدة من **وزارة العدل** و**الهيئة العامة للعقار**، تغطي **4.93 مليون** سجل عقاري عبر **159 ملف CSV** من 2018 إلى 2025.

البيانات العقارية السعودية متفرقة بين بوابات حكومية متعددة، بتنسيقات مختلفة وعناوين عربية وتغييرات غير موثّقة في الهيكل. هذا المشروع يجمعها في مكان واحد مع توثيق واضح، وسجل بيانات وصفية (metadata registry)، ومراجعات جودة، وتحليلات أولية — ليصبح العمل بها أسهل للباحثين والمحللين والمستثمرين والمطورين.

هذا مشروع تجريبي قيد التطوير. أستكشف ما يمكن أن تكشفه هذه البيانات من أنماط وعلاقات ورؤى. إذا وجدت فيه فائدة أو لديك أفكار، يسعدني أسمعها.

**كلمات مفتاحية:** بيانات عقارية سعودية, سوق العقار السعودي, صفقات وزارة العدل, مؤشرات الهيئة العامة للعقار, بيانات الإيجار, إيجار, البيانات المفتوحة السعودية, أسعار العقار في الرياض, Saudi Arabia real estate data, Saudi property market, MOJ transactions, REGA indicators, Saudi rental data, Ejar, open data Saudi Arabia

- **1.41 مليون** صفقة بيع عبر جميع المناطق الإدارية الـ 13، و175 مدينة، و13,398 حي

## سياق عام — كيف تعمل البيانات العقارية السعودية

في المملكة جهتان رئيسيتان تنشران بيانات عقارية:

**وزارة العدل** تسجّل جميع المعاملات العقارية عبر كتابات العدل. كل عملية بيع أو رهن أو حجز أو نقل ملكية أو تعديل صك تمر عبر الوزارة. هذه **بيانات على مستوى المعاملة الفردية** — سجلات تتضمن السعر الفعلي والمساحة والتاريخ ورقم المرجع. وهي المصدر الأغنى.

**الهيئة العامة للعقار** تنظّم السوق وتنشر **مؤشرات مجمّعة** — متوسط الأسعار لكل متر مربع، وأعداد الصفقات، ومقاييس الإيجار حسب المنطقة والربع. أقل تفصيلاً من وزارة العدل، لكنها تشمل بيانات الإيجار وتصنيفات نوع العقار التي لا تتوفر في بيانات الوزارة.

**وزارة الشؤون البلدية والقروية والإسكان** تدير التخطيط العمراني ورسوم الأراضي البيضاء وسياسات الإسكان. ليست مصدر بيانات في هذا المستودع، لكنها سياق مهم لأي تحليل تنظيمي.

جميع البيانات منشورة عبر [بوابة البيانات المفتوحة الوطنية](https://open.data.gov.sa)، وتخضع لرخصة البيانات المفتوحة السعودية. راجع [قاموس المصطلحات](GLOSSARY.md) لتعريفات المصطلحات العربية المستخدمة.

### وزارة العدل مقابل الهيئة العامة للعقار — ماذا يقدم كل منهما

| الجانب | وزارة العدل | الهيئة العامة للعقار |
|--------|------------|---------------------|
| **مستوى البيانات** | معاملات فردية | مؤشرات مجمّعة |
| **سعر فعلي لكل صفقة** | نعم | لا (متوسط/أدنى/أعلى للمتر) |
| **مساحة فعلية لكل صفقة** | نعم | لا (إجماليات لكل مجموعة) |
| **أرقام مرجعية** | نعم | لا |
| **بيانات إيجار** | لا | نعم (جميع المناطق الـ 13) |
| **نوع العقار (أرض/فيلا/شقة)** | فقط الربع 1-3 من 2023 | نعم (في ملفات المؤشرات) |
| **الفترة الزمنية** | 2020–2025 (مبيعات)، 2024–2025 (عمليات) | 2024–2025 (مبيعات)، 2019–2024 (إيجار) |
| **التغطية الجغرافية** | جميع المناطق الـ 13 | 6 مناطق (مبيعات)، 13 (إيجار) |

## القيود المعروفة

راجعنا البيانات بعمق ووثّقنا ما **لا تستطيع** تقديمه — لتوفير عليك اكتشاف نفس الطرق المسدودة:

1. **لا يمكن تتبع دورة حياة العقار.** أرقام المرجع خاصة بكل معاملة وليست بكل عقار. لا يمكنك تتبع عقار من الشراء إلى الرهن إلى الحجز إلى البيع. ([التفاصيل](docs/REFERENCE_NUMBER_AUDIT.md))

2. **لا يمكن التمييز بين الأرض والمبنى** إلا في الربع 1–3 من 2023 ومؤشرات الهيئة المجمّعة. عمود `تصنيف العقار` يحدد القطاع فقط (سكني/تجاري/زراعي) — وليس ما إذا كان أرضاً أو فيلا مبنية. ([التفاصيل](docs/ASSET_TYPE_AUDIT.md))

3. **ملفات هوية العقار هي سجلات خدمات** وليست جسراً بين المعاملات. رغم الاسم الواعد، تحتوي على قيم ثابتة للمنطقة والمدينة بدون أرقام صكوك أو قطع. ([التفاصيل](docs/PROPERTY_IDENTITY_AUDIT.md))

4. **36 صيغة كتابة لأسماء المناطق** — نفس المنطقة تظهر بأكثر من صيغة عربية عبر الملفات المختلفة. استخدم [`data/region_mapping.csv`](data/region_mapping.csv) للتوحيد قبل أي تجميع.

5. **عدم اتساق في صيغة التاريخ** — معظم الملفات تستخدم `YYYY/MM/DD`، وبعض ملفات 2024 الشهرية تستخدم `M/D/YYYY`. معظم الملفات تتضمن تاريخاً ميلادياً وهجرياً معاً.

6. **أرقام بفواصل** — حقول السعر والمساحة تستخدم فاصلة الآلاف (`1,800,000`). أزل الفواصل قبل أي عملية حسابية.

7. **فجوات في تغطية مبيعات الهيئة** — 6 مناطق فقط من 13 لديها بيانات مؤشرات مبيعات من الهيئة. 7 مناطق (عسير، القصيم، الجوف، الحدود الشمالية، نجران، تبوك، جازان) ليس لديها ملفات مبيعات من الهيئة. بيانات وزارة العدل تغطي جميع المناطق الـ 13.

8. **خلل في بيانات 2023** — انخفض حجم الصفقات إلى 140 ألف من 282 ألف في 2021. كذلك الربع الأول من 2023 له هيكل مختلف (13 عمود بدلاً من 10، يتضمن `نوع العقار`). سبب الانخفاض غير واضح — قد يكون تباطؤ فعلي في السوق أو تغيير في طريقة جمع البيانات.

| **مسار الحجوزات** — اتجاهات الحجز والإفراج والتنفيذ | عالية | عالي | متوسطة — مستثمرو الأصول المتعثرة |
| **تقدير العائد الإيجاري** — إيجارات الهيئة ÷ أسعار بيع الوزارة | متوسطة | عالي | عالية — المستثمرون الباحثون عن عائد |
| **الأنماط الموسمية** — مقارنة ربع بربع مع تعديل رمضان/الحج | عالية | متوسط | متوسطة — توقيت القرارات |
| **اتجاهات حجم الصفقات** — هل القطع تتقلص؟ الوحدات أصغر؟ | عالية | عالي | متوسطة — هيكل السوق |

## طرق الوصول للبيانات

### القراءة المباشرة من CSV

```python
import pandas as pd
from pathlib import Path

# تحميل جميع ملفات مبيعات وزارة العدل
frames = []
for f in sorted(Path("moj/sales").glob("MOJ-Sales-*.csv")):
    df = pd.read_csv(f)
    frames.append(df)
sales = pd.concat(frames, ignore_index=True)

# تنظيف عمود السعر (إزالة فاصلة الآلاف)
sales['السعر'] = sales['السعر'].astype(str).str.replace(',', '').astype(float)
sales['المساحة'] = sales['المساحة'].astype(str).str.replace(',', '').astype(float)
```

### توحيد أسماء المناطق

طبّق الربط قبل أي تجميع عبر الملفات:

```python
mapping = pd.read_csv("data/region_mapping.csv")
variant_to_canonical = dict(zip(mapping['variant'], mapping['canonical_en']))
sales['region_en'] = sales['المنطقة'].map(variant_to_canonical)
```

### استعلامات سجل البيانات الوصفية

سجل البيانات الوصفية يفهرس هيكل كل ملف CSV وأنواع الحقول والقيم الممكنة وعيّنات من الصفوف:

```bash
# إعادة بناء السجل بعد إضافة بيانات جديدة
python3 scripts/build_registry.py

# ما الملفات التي تحتوي على بيانات أسعار؟
sqlite3 registry.db "SELECT f.filename, fi.canonical_name
  FROM files f JOIN fields fi ON f.id = fi.file_id
  WHERE fi.canonical_name = 'price';"

# ما تصنيفات العقارات الموجودة؟
sqlite3 registry.db "SELECT ev.value, sum(ev.count)
  FROM enum_values ev JOIN fields f ON ev.field_id = f.id
  WHERE f.canonical_name = 'property_classification'
  GROUP BY ev.value ORDER BY sum(ev.count) DESC;"

# عرض جميع أسماء الحقول المعيارية
sqlite3 registry.db "SELECT DISTINCT canonical_name FROM fields
  WHERE canonical_name IS NOT NULL ORDER BY canonical_name;"
```

## الوضع القانوني

ملفات CSV هي **بيانات حكومية مفتوحة غير معدّلة** منشورة بموجب [رخصة البيانات المفتوحة السعودية](https://open.data.gov.sa/en/pages/policies/license).

**ملخص:**
- **إعادة التوزيع:** مسموح مع الإسناد
- **الأعمال المشتقة:** مسموح — *"حرية توزيع وإنتاج وتحويل الأعمال المتعلقة بمجموعات البيانات والبناء عليها"*
- **الاستخدام التجاري:** غير محظور صراحة (لا يوجد شرط غير تجاري)
- **الإسناد:** مطلوب — اذكر وزارة العدل / الهيئة العامة للعقار كمصدر البيانات
- **تحفّظ التعديل:** تشترط وزارة العدل أن تُحفظ بياناتها الخام *"بعدم تعديلها."* ملفات CSV في هذا المستودع غير معدّلة. التحليلات والأعمال المشتقة موسومة بوضوح كعمل خاص بنا وليست بيانات حكومية.

راجع [LICENSE-DATA-AR.md](LICENSE-DATA-AR.md) للتفاصيل الكاملة بما في ذلك الاقتباسات النصية من السياسات ومتطلبات الإسناد والتمييز بين البيانات الخام والمشتقة.
